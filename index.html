CSS Topic - Borders

Clipping a background to text is not supported in all browsers, but I'll show you how to do it below on most modern browsers using some special CSS properties that use vendor prefixes.

Choose a color

Next, I choose some text. I am using Google's Notable font in this example. Because I wanted the gradient to be the width of the text, I set the class to display: inline-block as a quick way to make the bounding box the same width of the text (by default, paragraph width is 100% of its bounding box, which, in this case, would be the entire screen). Since a few browsers do not support any form of background clipping on text, I set the text color to a fallback color.

Background clip!

Now comes the magic. CSS has a background-clip: text property, but it's still experimental. Some browsers don't support it (Firefox and Edge do, however, but don't even think about it working on Internet Explorer).

For Chrome and Safari browsers, you need to use what are called vendor prefixes. Adding -webkit-background-clip: text as a fallback below background-clip: text will enable background text clipping on Chrome and Safari and a few other lesser-used browsers. (Chrome, Safari, and those few other browsers use a rending engine called webkit which becomes the vendor prefix).

We need one more step. The browsers that support background-clip: text will clip the background to the text and the background will become the text fill. The webkit browsers have a strange quirk where the font color will override the background. To fix this, but still keep the font color for non-webkit browser that don't support background-clip: text, I use another vendor prefix property -webkit-text-fill-color: transparent that will make the text transparent only on webkit browsers.
